# LIVP Extractor 配置文件说明

## 概述

`config.py` 包含了 LIVP Extractor 的所有配置参数。通过修改这些参数,你可以调整程序的性能、内存使用和行为。

## 配置文件位置

```
tools/config.py
```

## 配置参数说明

### 并发配置

#### OUTER_MAX_WORKERS (外层工作线程数)
- **默认值**: 50
- **说明**: 同时处理多少个LIVP文件
- **影响**: 
  - 增加此值可以提高处理速度
  - 但会增加内存使用和CPU负载
- **建议**: 
  - 高性能机器: 50
  - 中等性能机器: 30
  - 低性能机器: 20

#### OUTER_CONCURRENT_FILES (外层并发文件数)
- **默认值**: 50
- **说明**: 同时处理的LIVP文件数量上限
- **影响**: 
  - 增加此值可以提高处理速度
  - 但会增加内存使用
- **建议**: 
  - 内存充足(16GB+): 50
  - 内存中等(8-16GB): 30
  - 内存不足(8GB以下): 20

#### INNER_MAX_WORKERS (内层工作线程数)
- **默认值**: 30
- **说明**: 处理单个LIVP文件内的HEIC/MOV文件的线程数
- **影响**: 
  - 增加此值可以提高单个LIVP文件的处理速度
  - 但会增加内存使用
- **建议**: 
  - 高性能机器: 30
  - 中等性能机器: 20
  - 低性能机器: 10

### 批次处理配置

#### BATCH_SIZE_MULTIPLIER (批次大小倍数)
- **默认值**: 4
- **说明**: 批次大小 = 线程数 × 倍数
- **影响**: 
  - 增加此值可以确保线程池始终有任务
  - 但会增加内存使用
- **建议**: 
  - 保持默认值4
  - 如果内存不足,可以降到2或3

#### BATCH_SIZE_MAX (批次大小上限)
- **默认值**: 100
- **说明**: 每个批次最多处理多少个任务
- **影响**: 
  - 增加此值可以提高并发效率
  - 但会增加内存使用
- **建议**: 
  - 保持默认值100
  - 如果内存不足,可以降到50

### I/O配置

#### FILE_COPY_BUFFER_SIZE (文件复制缓冲区大小)
- **默认值**: 8MB (8 * 1024 * 1024)
- **说明**: 文件复制时的缓冲区大小
- **影响**: 
  - 增加此值可以减少系统调用次数
  - 但会增加内存使用
- **建议**: 
  - 保持默认值8MB
  - 如果内存充足,可以尝试16MB
  - 不建议超过16MB,收益递减

### 内存配置

#### MEMORY_PER_FILE_MB (每个文件占用的内存)
- **默认值**: 200MB
- **说明**: 估算每个文件占用的内存
- **影响**: 
  - 用于动态调整并发数
  - 不需要手动修改
- **建议**: 
  - 保持默认值

#### MEMORY_USAGE_THRESHOLD (内存使用率阈值)
- **默认值**: 90%
- **说明**: 超过此阈值时降低并发数
- **影响**: 
  - 防止内存不足
  - 保护系统稳定性
- **建议**: 
  - 保持默认值90%
  - 如果内存充足,可以提高到95%

### 性能监控配置

#### ENABLE_PERFORMANCE_MONITORING (启用性能监控)
- **默认值**: True
- **说明**: 是否启用性能监控
- **影响**: 
  - 启用后会显示实时性能数据
  - 但会有少量性能开销
- **建议**: 
  - 调试时: True
  - 生产环境: False

#### PERFORMANCE_REPORT_INTERVAL (性能报告间隔)
- **默认值**: 5秒
- **说明**: 性能报告的输出间隔
- **影响**: 
  - 减少此值可以更频繁地看到性能数据
  - 但会增加I/O开销
- **建议**: 
  - 保持默认值5秒

### 日志配置

#### LOG_LEVEL (日志级别)
- **默认值**: INFO
- **可选值**: DEBUG, INFO, WARNING, ERROR, CRITICAL
- **说明**: 控制日志输出的详细程度
- **影响**: 
  - DEBUG: 最详细,包含所有调试信息
  - INFO: 一般信息
  - WARNING: 只显示警告和错误
  - ERROR: 只显示错误
  - CRITICAL: 只显示严重错误
- **建议**: 
  - 调试时: DEBUG
  - 正常使用: INFO
  - 生产环境: WARNING

#### REDUCE_VERBOSE_LOGGING (减少详细日志)
- **默认值**: True
- **说明**: 是否减少高频日志输出
- **影响**: 
  - 启用后会减少日志输出,提升性能
  - 但会丢失一些调试信息
- **建议**: 
  - 调试时: False
  - 正常使用: True

### 输出配置

#### EXPORT_TIMELINE_JPEG (导出时间轴JPEG)
- **默认值**: True
- **说明**: 是否导出时间轴JPEG图片
- **影响**: 
  - 启用后会生成JPEG缩略图
  - 但会增加处理时间
- **建议**: 
  - 需要时间轴: True
  - 不需要时间轴: False

#### TIMELINE_JPEG_QUALITY (时间轴JPEG质量)
- **默认值**: 85
- **范围**: 1-100
- **说明**: JPEG图片的质量
- **影响**: 
  - 质量越高,文件越大
  - 质量越低,文件越小
- **建议**: 
  - 保持默认值85
  - 如果需要更高质量: 90-95
  - 如果需要更小文件: 70-80

#### WEB_ASSET_PREFIX (Web资源前缀)
- **默认值**: /assets
- **说明**: Web资源路径的前缀
- **影响**: 
  - 用于生成时间轴JSON中的路径
- **建议**: 
  - 根据你的Web服务器配置调整

### 元数据配置

#### EXTRACT_GPS (提取GPS信息)
- **默认值**: True
- **说明**: 是否提取GPS信息
- **影响**: 
  - 启用后会提取GPS坐标
  - 但会增加处理时间
- **建议**: 
  - 需要GPS信息: True
  - 不需要GPS信息: False

#### EXTRACT_WEATHER (提取天气信息)
- **默认值**: True
- **说明**: 是否提取天气信息
- **影响**: 
  - 启用后会调用天气API
  - 但会增加处理时间和网络请求
- **建议**: 
  - 需要天气信息: True
  - 不需要天气信息: False

#### WEATHER_API_TIMEOUT (天气API超时)
- **默认值**: 10秒
- **说明**: 天气API请求的超时时间
- **影响**: 
  - 超时时间太长会影响处理速度
  - 超时时间太短可能导致请求失败
- **建议**: 
  - 保持默认值10秒

### 文件处理配置

#### SKIP_EXISTING_FILES (跳过已存在的文件)
- **默认值**: False
- **说明**: 是否跳过已存在的文件
- **影响**: 
  - 启用后会跳过已存在的文件
  - 但可能导致文件不是最新版本
- **建议**: 
  - 增量更新: True
  - 完全重新处理: False

#### OVERWRITE_EXISTING_FILES (覆盖已存在的文件)
- **默认值**: True
- **说明**: 是否覆盖已存在的文件
- **影响**: 
  - 启用后会覆盖已存在的文件
  - 但会增加处理时间
- **建议**: 
  - 需要最新文件: True
  - 保留已有文件: False

#### PRESERVE_ORIGINAL_FILENAME (保留原始文件名)
- **默认值**: True
- **说明**: 是否保留原始文件名
- **影响**: 
  - 启用后会保留原始文件名
  - 但可能导致文件名冲突
- **建议**: 
  - 保持默认值True

### 时间轴配置

#### GENERATE_TIMELINE (生成时间轴)
- **默认值**: True
- **说明**: 是否生成时间轴JSON
- **影响**: 
  - 启用后会生成时间轴JSON
  - 但会增加处理时间
- **建议**: 
  - 需要时间轴: True
  - 不需要时间轴: False

#### MERGE_TIMELINES (合并时间轴)
- **默认值**: True
- **说明**: 是否合并所有时间轴
- **影响**: 
  - 启用后会合并所有时间轴到一个文件
  - 但会增加处理时间
- **建议**: 
  - 需要合并时间轴: True
  - 不需要合并: False

### 错误处理配置

#### CONTINUE_ON_ERROR (遇到错误时继续)
- **默认值**: True
- **说明**: 遇到错误时是否继续处理其他文件
- **影响**: 
  - 启用后会跳过错误文件,继续处理
  - 但可能导致部分文件处理失败
- **建议**: 
  - 批量处理: True
  - 严格模式: False

#### MAX_RETRY_COUNT (最大重试次数)
- **默认值**: 3
- **说明**: 遇到错误时的最大重试次数
- **影响**: 
  - 增加重试次数可以提高成功率
  - 但会增加处理时间
- **建议**: 
  - 保持默认值3

#### RETRY_INTERVAL (重试间隔)
- **默认值**: 1秒
- **说明**: 重试之间的间隔时间
- **影响**: 
  - 增加间隔可以避免频繁重试
  - 但会增加处理时间
- **建议**: 
  - 保持默认值1秒

## 配置建议

### 高性能配置 (内存充足,16GB+)
```python
OUTER_MAX_WORKERS = 50
OUTER_CONCURRENT_FILES = 50
INNER_MAX_WORKERS = 30
FILE_COPY_BUFFER_SIZE = 16 * 1024 * 1024  # 16MB
```

### 平衡配置 (内存中等,8-16GB)
```python
OUTER_MAX_WORKERS = 30
OUTER_CONCURRENT_FILES = 30
INNER_MAX_WORKERS = 20
FILE_COPY_BUFFER_SIZE = 8 * 1024 * 1024  # 8MB
```

### 低内存配置 (内存不足,8GB以下)
```python
OUTER_MAX_WORKERS = 20
OUTER_CONCURRENT_FILES = 20
INNER_MAX_WORKERS = 10
FILE_COPY_BUFFER_SIZE = 4 * 1024 * 1024  # 4MB
```

## 注意事项

1. 修改配置后需要重启程序才能生效
2. 并发数设置过高可能导致内存不足
3. 缓冲区大小设置过大可能导致内存不足
4. 建议根据实际硬件配置调整参数
5. 如果遇到内存不足,建议降低并发数和缓冲区大小

## 故障排除

### 问题: 内存使用过高
**解决方案**: 
- 降低 OUTER_CONCURRENT_FILES
- 降低 INNER_MAX_WORKERS
- 降低 FILE_COPY_BUFFER_SIZE

### 问题: 处理速度慢
**解决方案**: 
- 增加 OUTER_CONCURRENT_FILES
- 增加 INNER_MAX_WORKERS
- 增加 FILE_COPY_BUFFER_SIZE
- 关闭性能监控 (ENABLE_PERFORMANCE_MONITORING = False)
- 减少日志输出 (LOG_LEVEL = "WARNING", REDUCE_VERBOSE_LOGGING = True)

### 问题: 程序崩溃
**解决方案**: 
- 降低并发数
- 降低缓冲区大小
- 检查系统日志
- 启用详细日志 (LOG_LEVEL = "DEBUG")
